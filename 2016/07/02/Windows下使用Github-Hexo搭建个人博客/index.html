标签： blog Hexo

摘要：这是我第一次搭建博客，也是我的第一篇教程。我想要把整个过程都记录下来，给需要搭建博客的同学一个参考。

前言

教程主要介绍了如何使用Github和Hex来搭建属于自己的博客。

为什么要使用Github+Hexo这样的组合来搭建博客？
很多人用wordpress,你为什么要用Github Pages来搭建？

  • Github有免费空间（官方没有磁盘限制，但是推荐1GB以下），资料自己管理，保存可靠；
  • 学习着用Github，享受Github的便利，上面有很多大牛，有很多项目，眼界会开阔；
  • 顺便可以了解Github工作原理，了解最好的团队协作流程；
  • Github现在是趋势；

什么是Hexo？为什么用Hexo？

  • Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
  • 引用下作者的话，使用Hexo是因为：
    不可思议的快速 ─ 只要一眨眼静态文件即生成完成
    支持 Markdown语法
    仅需一道指令即可部署到 GitHub Pages 和 Heroku
    已移植 Octopress 插件
    高扩展性、自订性
    兼容于 Windows, Mac & Linux

安装Hexo

hexo是基于node.js的静态博客，官网也是搭建在GitHub上

  • 在电脑上新建一个blog文件夹,该文件夹用于存放你的博客文件,然后右键单击选择“Git Bash”
  • 大家估计都有被“墙”的经历，安装hexo为了避免出现类似情况，我使用淘宝NPM镜像,输入以下命令等待安装完成

$ npm install -g cnpm –registry=https://registry.npm.taobao.org

  • 使用淘宝NPM安装Hexo

    $ cnpm install -g hexo-cli
    • 与原先的npm完全一样，只是命令改为cnpm,一样等待hexo安装完成

    出现的WARN可以不用理会
    继续输入以下命令

    $ cnpm install hexo –save
    安装完成后，在输入命令，验证是否安装正确

    $ hexo -v

配置SSH keys
我们如何让本地git项目与远程的github建立联系呢？用SSH keys。

  • 检查SSH keys的设置
    首先我们需要检查你电脑上现有的ssh key：

    $ cd ~/. ssh 检查本机的ssh密钥

如果提示：No such file or directory 说明你是第一次使用git。

  • 生成新的SSH Key：

    $ ssh-keygen -t rsa -C “你的邮件地址”

Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

注意1: 此处的邮箱地址，你可以输入自己的邮箱地址；注意2: 此处的「-C」的是大写的「C」

然后系统会要你输入密码：
Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>
在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
注意：输入密码的时候没有字样的，你直接输入就可以了。
最后看到这样的界面，就成功设置ssh key了：

*添加SSH Key到GitHub
在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。
1、因为是在家目录创建的ssh，所以可以通过在git输入

$ cd ~
$ pwd

来查看生成ssh的目录，然后打开/.ssh/id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
2、登陆github系统。点击右上角的 Account Settings—->SSH Public keys —-> add another public keys
3、把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了

测试
可以输入下面的命令，看看设置是否成功，git@github.com的部分不要修改：

ssh -T git@github.com

如果是下面的反馈：
The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?
不要紧张，输入yes就好，然后会看到：
Hi cnfeat! You’ve successfully authenticated, but GitHub does not provide shell access.
SSH Key配置成功

本机已成功连接到github。

本地运行hexo

  • 初始化hexo

    hexo init

  • 安装生成器

    cnpm install

  • 运行hexo,以后要在本地运行博客只要输入该命令即可

    hexo s -g

    打开浏览器，输入localhost:4000,就可以在本地看到你的个人博客了
    或者先生成web项目，输入：

    hexo g

启动服务器预览效果，输入：

hexo s -o

  • 停止运行
    按住Ctrl+C键即可停止

安装默认插件完成后，为了支持github，还需要安装另一个插件：

cnpm install hexo-deployer-git --save

完成后出现以下目录

这还只是本地项目，现在要发布到github上面去，我们需要配置项目目录下的_config.yml文件，使用使用Sublime_Text编辑器（绿色软件无需安装，解压即可使用）打开blog/_config.yml文件，进行配置，打开它找到最后一行，我们把github的配置信息填上去。这里ZhenjiangMeng是我的登录名，你只需要将它改成你github的登录名，原封不动地配置好就行了。
不少教程会教你像第二张图那样配置，新版本是不能那样配的，请注意。


• 我们开始发布博客吧！当前路径命令行下输入：

hexo d

按照提示输入github的登录名和密码，就可以看到成功信息。
稍等一会就可以输入网址xxx.github.io访问你的博客啦~~~

修改主题

想要修改主题可以到https://github.com/hexojs/hexo/wiki/Themes进行选择，点击后面蓝色字体Demo可以进行预览，点击前面蓝色字体的名字可以获取该主题的详细文件

看到喜欢的主题可以点击主题名称，获取详细信息，先点击Clone or download,然后可以获取该主题地址.该主题会被下载到/themes文件夹里

获取该主题地址之后，可以使用以下命令将该主题下载到本地

git clone 主题地址 themes/主题名字

比如我想下载air这个主题

git clone https://github.com/hustcer/hexo-theme-air.git themes/air

下载了主题之后如果要使用我们要对根目录的_config.yml文件进行配置，将themes改为你需要修改的主题名字，然后执行以下命令重新进行构建。

hexo clean 
hexo g
hexo s


构建完成后可以刷新localhost:4000查看效果

管理博客

配置信息
使用Sublime_Text编辑器（绿色软件无需安装，解压即可使用）打开blog/_config.yml文件，进行配置

title: Mzj_blog #博客名称

subtitle: 一天进步一点 #副标题

description: 记录生活点滴 #简介

author: ZhenjiangMeng #博客作者

language: zh-CN #博客语言

timezone: #时区

url: http://ZhenjiangMeng.github.io #博客地址,与申请的GitHub一致
root: /

permalink: :year/:month/:day/:title/ #博客链接格式
permalink_defaults:

source_dir: source
public_dir: public
tag_dir: tags
archive_dir: archives
category_dir: categories
code_dir: downloads/code
i18n_dir: :lang
skip_render:

new_post_name: :title.md # File name of new posts
default_layout: post
titlecase: false # Transform title into titlecase
external_link: true # Open external links in new tab
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
future: true
highlight:
enable: true
line_number: true
auto_detect: true
tab_replace:

default_category: uncategorized
category_map:
tag_map:

date_format: YYYY-MM-DD #日期格式
time_format: HH:mm:ss

per_page: 10 #分页，每页文章数量
pagination_dir: page

theme: landscape #博客主题

deploy: #发布设置
type: git

#ZhenjiangMeng改为你的github用户名
repository: https://github.com/ZhenjiangMeng/ZhenjiangMeng.github.io.git
branch: master

  • 写一篇文章
    输入创建文章命令，生成一个md文件(/blog/source/_posts/)

    hexo new “博客名字”

    用编辑器打开hello.md文件,编写完后保存，这里建议使用Cmd Markdown（https://www.zybuluo.com/cmd/）来进行博客编写，然后导出Markdown文件。因为编写博客时涉及到Markdown语法，而Cmd Markdown可以轻松搞定，而且可以一边编写一边预览，节省时间。

    编写完成后，使用以下命令同步到github，发布更新博客
    hexo d -g
    发布成功后，访问https://yourname.github.io看下成果

hexo命令

常用命令：
hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub

常用复合命令：
hexo deploy -g
hexo server -g

简写：
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
